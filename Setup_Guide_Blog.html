How to Run a Python App from Scratch (A Complete Beginner's Guide)

So you've been handed a really cool custom Python application - maybe an interface that reads sensor data, or an automation tool - but you have no idea how to run it. You look at your Windows desktop, and there is no "Python" in sight. Where do you even begin?

Don't worry! Running a Python script from scratch is much easier than it sounds.

In this blog post, we are going completely back to basics. We will pretend your computer has absolutely no developer tools installed, and walk you step-by-step through setting up Anaconda (a fantastic, beginner-friendly Python manager), preparing your digital environment, and finally launching that custom app!

Step 1: Downloading Anaconda 🐍

The very first thing we need is a program that understands the Python language. Rather than dealing with complex raw Python installations, we are going to use Anaconda. Anaconda is like a virtual toolbox that not only installs Python but also makes managing different projects incredibly easy, especially if you ever want to analyze data later!

  1. Open your favorite web browser and go to anaconda.com/download.
  2. Look for the download button for Windows and click it. It will download an .exe installer.
    (Note: This is a robust toolbox, so the download might be close to a gigabyte. Give it a few minutes!)

Step 2: Installing Anaconda 💻

Now that we have the installer, let's get it onto your machine.

  1. Double-click the downloaded .exe file to start the installation.
  2. Click Next, and then click I Agree to the terms.
  3. When it asks who to install it for, select Just Me (recommended). Click Next.
  4. It will show a default destination folder (usually somewhere in your C: drive). Leave it exactly as it is and click Next.
  5. On the Advanced Options screen, leave all the boxes at their default settings. We don't want to accidentally change how your Windows computer operates behind the scenes!
  6. Click Install. Grab a coffee or tea - this can take anywhere from 5 to 15 minutes to fully configure itself.

Step 3: Opening the "Anaconda Prompt" ⬛

In movies, you see hackers typing furiously into black screens with green text. We are going to do something similar - but much friendlier!

  1. Open your standard Windows Start Menu.
  2. Type Anaconda Prompt into the search bar.
  3. You will see an app with a black icon pop up. Click it!

A black window will open. Look closely at the very left side of the blinking cursor. You should see the word (base). This means Anaconda is successfully running its default environment!

Step 4: Creating a "Clean Room" For Your App 🧹

Imagine trying to bake a cake, make a pizza, and cook a soup all in the exact same bowl. It would be a disaster! In Python, different apps sometimes require different versions of tools. To keep things clean, we create a fresh "environment" - think of it as a virtual clean room just for this specific app.

In your Anaconda Prompt window, type the following text exactly as written and press Enter:

conda create -n my_app python=3.11

(Here, my_app is simply the name of our clean room, and python=3.11 tells Anaconda to put the 3.11 version of Python inside it).

After a few seconds, it will pause and ask: Proceed ([y]/n)?.
Type y and press Enter to confirm.

Step 5: Turning On the Environment 💡

Now that the room is built, we need to step inside it. Tell Anaconda to activate the environment by typing:

conda activate my_app

Press Enter. Look at the left side of your screen again - the (base) text has magically transformed into (my_app)! You are now inside your custom app environment.

Step 6: Navigating to Your Code Folder 📁

Your computer's command prompt needs to know where your Python application is saved. We have to steer it into the correct folder using the "Change Directory" (cd) command.

  1. Find the folder on your computer where your Python files are saved (my_script.py, requirements.txt, etc.).
  2. Click the top address bar in your Windows folder and copy the entire file path.
  3. In your Anaconda Prompt, type cd, press the Spacebar, type a quotation mark ", paste your folder path, and end with a quotation mark ".

It should look something like this:

cd "C:\Path\To\Your\Application\Folder"

Press Enter. The text on the screen will change to show you are now operating "inside" that folder.

Step 7: Reading the Instruction Manual (Dependencies) 📚

Many Python apps rely on "helper" code written by other people, like tools to draw interfaces or read hardware ports. These helpers are called dependencies. Luckily, developers usually leave a list of these helpers inside a file called requirements.txt.

We can tell Python to read this list and automatically download everything we need! Type this command and press Enter:

pip install -r requirements.txt

You'll see a bunch of downloading progress bars pop up on the screen. Let it run until it returns you to the normal blinking cursor. All the puzzle pieces are now officially in place.

Step 8: Liftoff! 🚀

You've built the environment, navigated to the folder, and gathered your dependencies. There is only one thing left to do.

Type the command to run your specific application file (in this case, my_script.py) and hit Enter:

python my_script.py

Suddenly, a brand new graphical window will spring to life on your desktop! You've successfully taken a raw script and built everything required to run it from absolute scratch.

Give yourself a massive pat on the back. Welcome to the world of Python! 🎉
